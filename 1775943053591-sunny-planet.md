## Issue 1: Login/Signup Not Working on Vercel (HIGH PRIORITY)

### Root Cause

In `login.html` lines 523-550, the form handlers treat `MutsAuth.login()` and `MutsAuth.signup()` as synchronous functions, but they return **Promises** (async). The code checks `result.success` immediately, which fails because `result` is a Promise object.

### Fix Required

Update the form handlers to use `.then()` for async handling:

**File:** `login.html`

**Current (broken):**
```javascript
document.getElementById('login-form').onsubmit = function(e) {
    e.preventDefault();
    var result = window.MutsAuth.login(...);
    if (result.success) {  // BUG: result is a Promise!
        window.location.href = 'pages/dashboard/index.html';
    }
    else alert('AUTH_FAILED: ' + result.message);
};
```

**Fix:**
```javascript
document.getElementById('login-form').onsubmit = function(e) {
    e.preventDefault();
    window.MutsAuth.login(
        document.getElementById('login-email').value, 
        document.getElementById('login-password').value
    ).then(function(result) {
        if (result.success) {
            window.MutsTour.show();
            window.location.href = 'pages/dashboard/index.html';
        } else {
            alert('AUTH_FAILED: ' + result.message);
        }
    }).catch(function(err) {
        alert('AUTH_FAILED: ' + err.message);
    });
};
```

**Same fix needed for signup form (line 533-550)**

## Issue 3: Dynamic Content Management (HIGH PRIORITY)

### Current State

The mock data in services (hotels-service.js, listings-service.js, etc.) is hardcoded and used for development only. The manager should be able to create/edit hotels, packages, tours, etc. and have them appear on the main site.

### Architecture Design

The system should support **both API and offline modes**:

**A. When API is Connected:**
1. Manager creates content → POST to API endpoint (e.g., `/api/manager/content/hotels`)
2. API stores content in database
3. Main site fetches from API endpoints (e.g., `/api/hotels`)
4. Main site periodically polls for updates OR can be manually refreshed

**B. When API is NOT Connected (Offline):**
1. Manager creates content → stored in localStorage (`muts_manager_content`)
2. Main site fetches from localStorage as fallback

### Implementation Plan

**1. Main Site Services (Hybrid Architecture)**

Each service should check in this order:
```javascript
// Example: hotels-service.js
getAll: function() {
    // 1. Try API first if connected
    if (this.isAPILive()) {
        return this.fetchFromAPI('/hotels')
            .then(function(response) {
                // Cache locally for offline use
                this._cacheContent(response.data || response);
                return response.data || response;
            }.bind(this));
    }
    
    // 2. Check localStorage for manager-created content
    var localContent = this._getManagerContent();
    if (localContent && localContent.hotels && localContent.hotels.length > 0) {
        return Promise.resolve(localContent.hotels);
    }
    
    // 3. Fallback to mock data (development only)
    return Promise.resolve(this.mockData);
},

_cacheContent: function(data) {
    try {
        localStorage.setItem('muts_hotels_cache', JSON.stringify(data));
    } catch (e) {}
},

_getManagerContent: function() {
    try {
        return JSON.parse(localStorage.getItem('muts_manager_content'));
    } catch (e) {
        return null;
    }
},

isAPILive: function() {
    return window.MutsAPIConfig && window.MutsAPIConfig.isConnected();
}
```

**2. Refresh/Poll Mechanism**

Add ability to refresh content from API:
```javascript
// Add to each service
refresh: function() {
    if (this.isAPILive()) {
        return this.getAll(); // Will fetch fresh from API
    }
    return Promise.resolve(); // Nothing to refresh offline
}
```

For the UI, add a refresh button or auto-refresh interval (e.g., every 5 minutes).

**3. Manager Service - Content Publishing**

When manager saves content:
```javascript
// In manager-service.js - after successful save
publishContent: function(type, data) {
    if (this.isAPILive()) {
        // POST to API - backend stores in database
        return this.fetch('/content/' + type, {
            method: 'POST',
            body: JSON.stringify(data)
        });
    }
    
    // Offline fallback - store in localStorage
    var content = JSON.parse(localStorage.getItem('muts_manager_content') || '{}');
    if (!content[type]) content[type] = [];
    
    // Add or update item
    var idx = content[type].findIndex(function(item) { return item.id === data.id; });
    if (idx >= 0) {
        content[type][idx] = data;
    } else {
        data.createdAt = new Date().toISOString();
        content[type].push(data);
    }
    
    localStorage.setItem('muts_manager_content', JSON.stringify(content));
    return Promise.resolve({ success: true, offline: true });
}
```

**4. Content Keys Structure**

```javascript
// localStorage: muts_manager_content
{
    hotels: [{ id, name, tier, location, price, ... }],
    destinations: [{ id, name, type, region, ... }],
    tours: [{ id, name, duration, price, ... }],
    packages: [{ id, name, duration, price, ... }],
    blogs: [{ id, title, category, ... }],
    products: [{ id, name, category, price, ... }]
}
```
