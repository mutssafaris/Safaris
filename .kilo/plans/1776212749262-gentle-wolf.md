# Login Page Background Transformation Plan

## Goal
Transform the login page so that the safari image becomes a CSS background layer (secondary background) with a medium overlay for contrast, while all UI components sit on top.

## Current State
- Background: CSS gradients only (purple → magenta → dark)
- Image: HTML `<img>` element `.hero-character` positioned absolutely in `.hero-character-panel`
- Components: All UI elements on top (correct z-index)

## Target State
- Background layer 1 (bottom): Deep purple gradient base
- Background layer 2 (middle): Safari image centered/cover
- Background layer 3 (top): Dark overlay at 60-70% for contrast and text readability
- All UI components on top

## Implementation Steps

### Step 1: Modify `.landing-hero` CSS
Add the image as CSS background with blend modes:
```css
.landing-hero {
    background: 
        /* Top layer: dark overlay at 60-70% for text readability */
        linear-gradient(135deg, rgba(9, 0, 26, 0.65) 0%, rgba(26, 0, 46, 0.65) 50%, rgba(58, 0, 80, 0.65) 100%),
        /* Middle layer: safari image centered */
        url('images/hero/safari-guide.jpg') center center / cover no-repeat,
        /* Bottom layer: gradient base */
        linear-gradient(135deg, #09001a 0%, #1a002e 50%, #3a0050 100%);
    background-blend-mode: overlay, normal, normal;
}
```

### Step 2: Remove HTML Image Element
Remove the `<img class="hero-character">` from `login.html` inside `.hero-character-panel`:
```html
<!-- Remove this line -->
<img src="images/hero/safari-guide.jpg" alt="Safari Guide" class="hero-character">
```

### Step 3: Clean Up Unused CSS
- Remove `.hero-character` styles (no longer needed)
- Remove `.hero-character-panel` height and positioning (can simplify to just hold floating stats)
- Remove `.hero-character` glow animation keyframes (no longer needed)
- Keep `.floating-stat` styles - these remain as overlay elements

### Step 4: Position Floating Stats Correctly
The floating stats (MAASAI MARA, EXPLORE NOW) will need slight adjustment since they're currently positioned relative to the removed image. Update positions to:
- `top-right`: `top: 15%, right: 30%`
- `bottom-left`: `bottom: 20%, right: 40%`

## Files to Modify
1. `css/login-landing.css` - Update `.landing-hero` background and remove unused `.hero-character` styles
2. `login.html` - Remove the `<img class="hero-character">` element

## Result
- Safari image as centered background (layer 2)
- Dark gradient overlay at ~65% opacity for text contrast (layer 3)
- No animation (static gradient)
- All UI components remain on top
- Floating stat badges preserved with adjusted positions
